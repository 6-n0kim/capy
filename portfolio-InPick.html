InPick — AI 기반 여행 일정 생성 및 추천 시스템

사용자의 여행지, 동행자, 테마, 위치 정보를 기반으로 개인화된 일정을 생성하고 다양한 AI 추천 알고리즘을 활용한 장소 추천 서비스

InPick은 AI 기반 여행 일정 생성 및 추천 시스템으로, 사용자의 여행 선호도를 분석하여 개인화된 여행 일정을 자동 생성합니다. 콘텐츠 기반 추천, 협업 필터링, 시퀀스 기반 Next-POI, RAG(Vector DB)를 활용한 다양한 추천 방식을 제공합니다.

시스템 아키텍처

InPick은 Backend (FastAPI)와 Frontend (React)로 구성된 풀스택 애플리케이션입니다:

  • Backend (FastAPI): 여행 일정 관리, AI 추천 시스템, OAuth 인증, 지도 API 연동
  • Frontend (React): 사용자 인터페이스, 4단계 여행 계획 마법사, 지도 시각화

주요 기능

인증 시스템

  • 소셜 로그인: Google / Naver / Kakao OAuth 2.0 인증
  • 쿠키 기반 세션 관리 (MongoDB, TTL: 1800초)
  • 세션 자동 재활성화 (매 요청마다 갱신)
  • 계정 삭제 (soft delete 지원)

여행 일정 관리

  • 4단계 마법사로 일정 생성 (기본 정보 → 장소 선택 → 숙소 선택 → 최종 확인)
  • 일정 CRUD 작업 (생성, 조회, 수정, 삭제)
  • 일정 내 세부 장소 관리 (itinerary_item)
  • Kakao Map 기반 경로 시각화

AI 추천 시스템

  • CBF (Content-Based Filtering): 카테고리, 테마, 설명 기반 코사인 유사도 추천
  • CF (Collaborative Filtering): ALS 알고리즘 기반 사용자-아이템 행렬 분해
  • Next-POI 추천: GRU4Rec, SASRec 모델을 활용한 시퀀스 기반 다음 장소 예측
  • RAG 기반 추천: FAISS 벡터 DB를 활용한 위치·테마·동행자 조건 검색
  • GPT 추천: OpenAI, Claude를 활용한 자연어 기반 맞춤형 일정 추천

지도 기능

  • Kakao Maps SDK 통합 (지도 렌더링, 마커, 인포윈도우)
  • TMAP API (자동차 경로 탐색)
  • 장소 타입별 마커 아이콘 (음식점, 카페, 관광지 등)

관리자 기능

  • 대시보드 통계 (Recharts)
  • 사용자 관리, 카테고리 관리, 숙소 관리, SNS 계정 관리
  • 활동 로그 관리 (모든 주요 이벤트 로깅)

기술 스택

Backend (FastAPI)

  • Language: Python
  • Framework: FastAPI
  • Database: PostgreSQL, MongoDB
  • Vector DB: FAISS
  • Machine Learning: PyTorch, Implicit (ALS)
  • LLM/AI: OpenAI, Claude (via LangChain)
  • Auth: OAuth 2.0 (Google, Naver, Kakao)
  • Map API: Kakao Map, TMAP API
  • Infra: Docker, Jenkins, Gunicorn
  • Env: Conda (environment.yml)

Frontend

  • Language: TypeScript
  • Framework: React 19
  • Build Tool: Vite 7
  • Routing: React Router DOM 7
  • State Management: React Hooks (useState, useEffect)
  • UI Library: Radix UI, shadcn/ui
  • Styling: Tailwind CSS 4
  • Forms: React Hook Form, Zod
  • HTTP Client: Axios
  • Maps: Kakao Maps SDK
  • Charts: Recharts
  • Notifications: React Toastify
  • Infra: Docker, Jenkins, Nginx

추천 시스템 상세

1. 콘텐츠 기반 추천 (CBF)

  • 경로: app/ml/content_based_filtering/main.py
  • 카테고리, 테마, 설명 기반 코사인 유사도 계산
  • 장소 간 유사도 기반 추천

2. 협업 필터링 (CF - ALS)

  • 경로: app/ml/collaborative_filtering/main.py
  • 사용자-아이템 행렬 분해를 통한 잠재 요인 추출
  • Implicit 라이브러리를 활용한 ALS 알고리즘 구현

3. 시퀀스 기반 Next-POI 추천

  • GRU4Rec: app/ml/next_poi_gru4rec.py
  • SASRec: app/ml/next_poi_sas_rec.py
  • PyTorch 기반 딥러닝 모델로 사용자 방문 시퀀스 학습
  • 다음 방문할 장소 예측

4. RAG 기반 장소 검색

  • 경로: app/rag/faiss_store/
  • FAISS 벡터 DB를 활용한 유사도 검색
  • 위치, 테마, 동행자 조건을 자연어로 검색
  • LangChain을 통한 LLM 연동

인증 플로우

사용자 → /auth/{provider}/login → OAuth Provider 인증 → /auth/{provider}/callback → 사용자 정보 검증 → MongoDB 세션 생성 (TTL: 1800초) → app_user 테이블 등록/갱신 → 쿠키 기반 세션 관리

  • 세션 재활성화: 매 요청마다 자동 갱신 (UserLogMiddleware)
  • 로그아웃: GET /auth/logout (세션 삭제)
  • 계정 삭제: DELETE /auth/user (soft delete)

여행 계획 마법사 (4단계)

  • Step 1: 기본 정보 입력 - 지역, 기간, 동반자, 테마 선택
  • Step 2: 장소 선택 및 추천 - AI 추천 모델 선택, 장소 검색 및 선택
  • Step 3: 숙소 선택 - 숙소 검색 및 선택
  • Step 4: 최종 확인 - 일정 최종 확인 및 저장

데이터베이스 구조

PostgreSQL

  • app_user (유저 계정, 소셜 로그인, soft delete 지원)
  • place (장소: 관광지, 음식점, 카페 등)
  • accommodation (숙소 정보)
  • itinerary (여행 일정: 시작/종료일, 구성원, 테마 등)
  • itinerary_item (일정 내 세부 장소)
  • category (장소 카테고리)
  • region (지역 정보, 위경도 포함)
  • place_category (장소-카테고리 연결, 다대다)
  • place_registrar (사용자 등록 장소, 다대다)

MongoDB

  • 세션 저장소 (TTL: 1800초)
  • 활동 로그 저장

프론트엔드 주요 기능

  • 상태 관리: React Hooks 기반 (useState, useEffect), localStorage로 사용자 정보 캐싱
  • UI 컴포넌트: shadcn/ui 기반 커스텀 컴포넌트 (Card, Dialog, Badge, Button, Input, Switch)
  • 지도 통합: Kakao Maps SDK를 통한 지도 렌더링, 마커, 경로 표시
  • 폼 관리: React Hook Form + Zod를 통한 폼 검증 및 데이터 처리
  • API 클라이언트: Axios를 통한 백엔드 API 통신, 쿠키 기반 세션 관리

프로젝트 특징

  • 다양한 추천 알고리즘: CBF, CF, 시퀀스 기반, RAG 등 5가지 추천 방식 제공
  • 하이브리드 데이터베이스: PostgreSQL (정형 데이터) + MongoDB (세션, 로그)
  • 벡터 검색: FAISS를 활용한 고속 유사도 검색으로 RAG 구현
  • 딥러닝 모델: PyTorch 기반 GRU4Rec, SASRec 모델로 시퀀스 기반 추천
  • 소셜 로그인: Google, Naver, Kakao OAuth 2.0 통합 인증
  • 지도 API 통합: Kakao Maps, Naver Directions, TMAP API 연동
  • 타입 안정성: TypeScript를 전면 적용하여 타입 안전성 확보

프로젝트 경험

팀장으로서 전체 일정과 개발 방향을 총괄하며 팀원 간 협업을 조율했습니다. 데이터 구조의 일관성과 효율성을 확보하기 위해 데이터베이스를 직접 설계하고 서비스 전반의 데이터 흐름을 관리했습니다. 또한 추천 시스템의 핵심인 콘텐츠 기반 필터링(Content-Based Filtering)과 협업 필터링(ALS) 모델을 구현하여 사용자 맞춤형 여행지 추천 기능을 완성했습니다. 모델 성능을 개선하기 위해 다양한 하이퍼파라미터를 실험했으나, 기대한 수준의 평가 점수를 달성하지 못한 점은 아쉬움으로 남습니다. 이번 프로젝트를 통해 문제 해결 능력과 팀 리딩 역량을 강화했으며, 다양한 AI 모델을 실제 서비스에 적용하며 실무형 개발 역량을 한층 향상시킬 수 있었습니다.
