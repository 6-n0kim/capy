Fit Harmony — AI 기반 실시간 자세 교정 및 운동 추천 서비스

웹캠을 통해 사용자의 운동 자세를 실시간으로 분석하고 피드백을 제공하는 서비스입니다.

1. 시스템 아키텍처 (System Architecture)

프론트엔드에서 AI 모델을 직접 실행(On-Device AI)하여 서버 부하를 줄이고 실시간성을 확보했습니다.

2. 주요 기능 및 화면 (UI/UX)

인바디 데이터 시각화

Recharts 라이브러리를 활용하여 인바디 데이터의 변화 추이를 라인/바/레이더 차트로 직관적으로 보여줍니다.

OCR 인식

Tesseract OCR을 활용하여 골격근량, 나이, 체성분량, 성별을 인식하고 저장합니다.

3. 실시간 자세 교정 알고리즘 Flow

웹캠 입력부터 오차 계산, 피드백 출력까지의 논리적인 순서입니다.

sequenceDiagram
    participant User
    participant Webcam
    participant TFJS as PoseNet Model
    participant Analyzer as Angle Analyzer
    participant Feedback as Feedback Logic

    User->>Webcam: 1. 운동 시작
    Webcam->>TFJS: 2. 비디오 프레임 전달 (30fps)
    TFJS->>Analyzer: 3. 관절 키포인트 추출 (X, Y, Z, Score)
    
    rect rgb(240, 248, 255)
    Analyzer->>Analyzer: 4. 기준 각도와 현재 각도 계산 (Cosine Similarity)
    Analyzer->>Feedback: 5. 오차율(%) 전달
    Feedback->>Feedback: 6. 오차율에 따른 메시지 결정
    end
    
    Feedback->>User: 7. "무릎이 너무 나갔어요!" (음성/텍스트)
    Feedback->>TFJS: 8. 다음 프레임 대기

4. 데이터베이스 설계 (ERD)

사용자, 운동 정보, 기록 데이터를 중심으로 설계된 관계형 데이터베이스(RDB) 구조입니다.

▲ MySQL 주요 엔티티 관계도

6. 트러블 슈팅

API 호출 시 JSON 데이터가 정상적으로 전달되지 않는 문제를 해결하기 위해 Axios 인터셉터를 사용하여 요청/응답을 로깅하고, 데이터를 확인
JWT 인증 과정에서 토큰이 정상적으로 전달되지 않는 문제를 해결하기 위해 interceptor를 사용하여 토큰을 추가
CORS 문제를 해결하기 위해 프론트 작업을 하는 동기와 정보 공유 후 해결
